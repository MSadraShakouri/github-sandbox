# 📥 دانلود فایل‌ها از طریق پیام کامیت

یک workflow از GitHub Actions که به شما اجازه می‌دهد فقط با نوشتن یک پیام کامیت خاص، فایل‌ها را در مخزن خود دانلود کنید — بدون نیاز به ترمینال یا خط فرمان.

---

## ⚙️ راه‌اندازی

0. این مخزن را fork کنید
1. به مخزن خود در GitHub بروید
2. روی **Settings** → **Actions** → **General** کلیک کنید
3. به بخش **Workflow permissions** بروید
4. گزینه **Read and write permissions** را انتخاب کرده و **Save** را بزنید

کار تمام است — نیازی به توکن یا سکرت نیست.

---

## 🚀 طریقه استفاده

با ویرایش هر فایل مستقیماً در GitHub و استفاده از یک پیام کامیت خاص هنگام ذخیره، دانلودها را فعال می‌کنید.

### چگونه یک دانلود را فعال کنید

1. هر فایلی را در مخزن خود در GitHub باز کنید (مثلاً همین `README.md`)
2. روی آیکون مداد (✏️) در بالا سمت راست کلیک کنید تا آن را ویرایش کنید
3. یک تغییر کوچک ایجاد کنید (یک فاصله، یک خط خالی، هر چیزی)
4. به بخش **Commit changes** بروید
5. گزینه **Commit directly to the `main` branch** را انتخاب کنید
6. در جعبه پیام کامیت، یکی از دستورات زیر را تایپ کنید
7. روی **Commit changes** کلیک کنید

workflow به طور خودکار اجرا می‌شود و فایل‌های دانلود شده در پوشه `downloads/` ظاهر می‌شوند.

---

## 📝 دستورات

### دانلود فایل‌ها به صورت جداگانه

هر فایل را دانلود کرده و با نام اصلی خود ذخیره می‌کند.

```
download: URL1 URL2 URL3
```

**مثال‌ها:**

```
download: https://example.com/file.zip
```

```
download: https://example.com/a.zip https://example.com/b.pdf https://example.com/c.zip
```

---

### دانلود و بسته‌بندی در یک فایل ZIP

تمام فایل‌ها را دانلود کرده و در یک بایگانی `.zip` با زمان‌بندی ذخیره شده در `downloads/` قرار می‌دهد.

```
download-zip: URL1 URL2 URL3
```

**مثال‌ها:**

```
download-zip: https://example.com/file.zip
```

```
download-zip: https://example.com/a.zip https://example.com/b.pdf https://example.com/c.zip
```

بایگانی حاصل به این صورت نامگذاری می‌شود: `archive_20250423_153012.zip`

---

## 📁 خروجی

| دستور | نتیجه |
|-------|-------|
| `download:` | هر فایل به صورت جداگانه در `downloads/` با نام اصلی خود ذخیره می‌شود |
| `download-zip:` | تمام فایل‌ها در یک بایگانی `archive_YYYYMMDD_HHMMSS.zip` داخل `downloads/` قرار می‌گیرند |

---

## 👀 بررسی نتیجه

پس از کامیت، می‌توانید workflow را نظارت کنید:

1. روی برگه **Actions** در مخزن خود کلیک کنید
2. روی آخرین اجرای workflow کلیک کنید تا پیشرفت و لاگ‌ها را ببینید
3. پس از اتمام، به برگه **Code** برگردید و پوشه `downloads/` را باز کنید تا فایل‌های خود را بیابید

---

## ⚠️ نکات

- آدرس‌های اینترنتی باید قابل دسترسی عمومی باشند (بدون نیاز به ورود)
- آدرس‌های متعدد را با فاصله از هم جدا کنید
- workflow با استفاده از `[skip ci]` در پیام کامیت خود، از اجرای مجدد خود جلوگیری می‌کند تا از حلقه بی‌نهایت جلوگیری شود
- اگر هیچ دستور معتبر `download:` یا `download-zip:` در پیام کامیت پیدا نشود، workflow بدون انجام کاری خارج می‌شود
